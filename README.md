# WIT 战斗管理器 — Walk In the Terra

专为 **Walk In the Terra (WIT)** 明日方舟跑团规则设计的战斗管理工具，帮助主持人追踪战斗中的角色状态、伤害、元素损伤和回合顺序，节省玩家记忆空间。

## 快速启动

```bash
pip install tkinter  # 通常 Python 自带，无需安装
python main.py
```

程序关闭时自动保存所有数据到 `data.json`，下次启动自动恢复。

## 功能概览

### 角色管理
- 玩家 / 怪物统一管理，同一列表显示，类型列 + 颜色区分（蓝=玩家，红=怪物）
- 属性：HP（含临时HP）、速度、重量、物抗、法抗、护甲类型、元素韧性、精英化阶段
- 37 种状态效果（17 正面 BUFF + 20 负面 BUFF），含复选框编辑

### 先攻系统
| 模式 | 说明 |
|------|------|
| **传统先攻** | 每单位掷 d20 + 速度，按检定值降序固定整场顺序，平局→比速度→重掷 |
| **团队先攻** | 每阵营取速度最高+最低求和，高者阵营先动 |
| **客观判断** | GM 手动指定先动阵营 |

### 伤害与治疗
- 物理 / 法术 / 真实伤害，自动计算抗性减免、护盾抵消、脆弱加成
- 治疗系统，禁疗 / 亲和 状态自动处理
- GM 直接输入骰娘生成的数值即可

### 元素损伤系统
- 6 种损伤类型：凋亡 / 组织 / 毒性 / 侵蚀 / 灼燃 / 神经
- 元素韧性追踪（精零=6 / 精一=9 / 精二=12）
- 韧性归零自动触发元素爆发，施加对应状态并提示 GM 输入伤害

### 状态效果系统
- **带 X 计数器**：护盾、屏障、抵抗、元素屏障、伤害强化、脆弱、失重 — 支持叠层
- **升级链**：麻痹→眩晕、寒冷→冻结、困倦→睡眠、停顿→束缚
- **标记**特殊处理：同时视为停顿/震颤/寒冷/困倦，触发四路升级
- 按结束条件自动清理：回合结束 / 攻击后 / 受到治疗后 / X 归零

### 回合追踪
- **Turn**：场上所有单位行动一次后 +1
- **Now**：指示当前执行单位
- 回合结束时自动清理状态 + 恢复元素韧性
- 支持 迅捷 / 迟缓 回合重排序

## 项目结构

```
├── main.py              # 入口
├── models.py            # Unit / CombatState 数据模型 + 状态常量表
├── combat.py            # 伤害 / 治疗 / 元素损伤 / 状态升级 / 回合管理
├── persistence.py       # JSON 持久化
├── ui/
│   ├── main_window.py   # 主窗口（左右分栏 + 战斗日志）
│   ├── unit_panel.py    # 单位列表 + 筛选 + 详情
│   ├── unit_dialog.py   # 添加/编辑弹窗
│   └── combat_panel.py  # 先攻 / 伤害输入 / 元素损伤 / 状态操作
├── 工作资料/            # 规则参考文件（不上传）
│   ├── buff.txt
│   └── 元素损伤.txt
└── README.md
```
